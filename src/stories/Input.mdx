# Input

O componente Input é um campo de entrada de dados versátil e acessível para formulários, com suporte a múltiplas variantes visuais, estados de validação e ícones.

## Visão Geral

O Input fornece uma interface consistente para coleta de dados do usuário, com foco em acessibilidade, usabilidade e customização. É ideal para uso em formulários, sistemas de busca e qualquer cenário que requeira entrada de dados.

## Variantes Visuais

O Input oferece três variantes visuais para diferentes contextos de design e necessidades de UI:

### Quando usar cada variante:

- **Outlined** (Padrão): Ideal para a maioria dos formulários, oferece boa hierarquia visual
- **Filled**: Para interfaces mais densas ou quando se deseja um visual mais "preenchido"
- **Transparent**: Perfect para overlays, headers ou situações onde o fundo não deve interferir

## Inputs com Ícones

Suporte a ícones no início (leading) e final (trailing) do campo para fornecer contexto adicional e melhorar a usabilidade:

### Casos de uso comuns:

- **Leading Icon**: Identificação do tipo de dado (User, Mail, Phone, etc.)
- **Trailing Icon**: Ações (Eye para senha, Search, Clear, etc.)
- **Ambos**: Estados complexos (User + Check para verificação)

<Canvas of={InputStories.WithIcons} />

## Estados Interativos

Diferentes estados para cobrir todos os cenários de interação do usuário:

### Comportamento dos estados:

- **Normal**: Campo interativo e editável
- **Disabled**: Não interativo, para ações não disponíveis
- **Required**: Indica obrigatoriedade com indicador visual
- **Read-only**: Exibe valor sem permitir edição
