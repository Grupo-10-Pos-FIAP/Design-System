# Design System

Este manual fornece diretrizes comuns para todos os profissionais envolvidos no desenvolvimento, manutenção e evolução de aplicações web. 
O objetivo é garantir uma identidade visual consistente de acordo com os padrões organizacionais, padrões de qualidade e políticas institucionais.

Os tokens e ícones do Design System podem ser aplicados em várias tecnologias web e mobile. 
Consulte a documentação respectiva para implementações específicas.

**Versão:** {packageJson.version}

## Fundamentos

Além de garantir que os componentes atendam a todas as diretrizes de documentação estabelecidas, esta biblioteca é construída sobre princípios fundamentais:

- **Manutenibilidade:** Código limpo, bem estruturado e compreensível que facilita encontrar, modificar e adicionar novos recursos — mesmo para desenvolvedores novos no projeto.
- **Escalabilidade:** Uma arquitetura que suporta a fácil incorporação de novas funcionalidades, suporte e componentes.
- **Acessibilidade:** Componentes que garantem navegação via mouse, toque e leitores de tela, atendendo às especificações de design que abordam outros critérios de acessibilidade.

## Atualizações da Biblioteca

A biblioteca recebe atualizações frequentes para adicionar recursos, melhorar o design dos componentes ou corrigir problemas. 
Durante o desenvolvimento, você pode notar que os componentes no layout ou Storybook são mais recentes do que os do seu projeto.

Como a biblioteca é instalada via NPM, atualizações automáticas não são recomendadas (exceto para correções de bugs). 
Os desenvolvedores devem garantir que estão usando a versão mais recente ao trabalhar com componentes.

Cada projeto ou equipe deve estabelecer políticas documentadas definindo quem pode realizar atualizações, quando e como.

## Melhores Práticas & Recomendações

- **Não sobrescreva os estilos dos componentes externamente.** Personalize os componentes apenas através de suas propriedades disponíveis, 
alterações de tema ou ajustes aprovados na biblioteca. Sobrescritas externas de estilo podem quebrar durante atualizações quando estruturas internas mudarem.

- **Sempre verifique os tipos e usos das propriedades dos componentes.** Cada componente possui documentação listando todas as propriedades, 
incluindo as obrigatórias. A tipificação especifica o comportamento visual e interativo, garantindo que os componentes funcionem como esperado e reduzindo problemas durante atualizações. 
Mesmo que um componente funcione sem uma propriedade obrigatória em uma versão, ele pode falhar em atualizações futuras.
